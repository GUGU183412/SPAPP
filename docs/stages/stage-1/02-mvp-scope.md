# 交付物 2：核心功能定义与 MVP 范围

## 1. 文档目的

本文件用于定义阶段一收敛后的 MVP 核心链路、功能优先级和明确不做项，作为阶段二 PoC 与阶段三 MVP 实现范围基线。

## 2. MVP 核心链路

当前 MVP 以最短问题解决闭环为核心：

`Landing Page -> Step 1 -> Step 2 -> Step 3 -> Feedback -> Unresolved Reason`

其业务目标不是构建完整训练产品，而是帮助用户快速确认：

- 是否已经正确锁死
- 是否可以信任缝线安全性
- 是否会伤门 / 是否装在正确位置
- 如果未解决，具体原因是什么

## 3. Must / Should / Won't

| 优先级 | 功能模块 | 详细描述 |
|---|---|---|
| Must | 防呆安装 GIF | 通过正反对比展示错误穿法与正确锁死穿法，解决压扣打滑 |
| Must | 安全承诺卡片 | 通过缝线工艺特写与静态承重信息回应安全性恐慌 |
| Must | Resolved 闭环 | 在教程后收集 Yes / No，并对未解决原因继续追问 |
| Should | 真人语音旁白 | 补足无声演示理解成本，帮助用户确认锁死动作与提示声 |
| Should | 康复示范入口 | 提供 1 个基础肩袖康复动作，建立专业信任感 |
| Won't | AI 视频诊断 | 当前阶段不做图像识别和开放式诊断 |
| Won't | 社交 / 打卡 | 当前阶段不建立账号体系，不做长期留存产品结构 |

## 4. 页面能力边界

### 4.1 当前必须覆盖

- 型号确认
- 安装对错判断
- 安全背书
- 门板保护与安装位置确认
- 是否解决反馈
- 未解决原因回收

### 4.2 当前不追求

- 内容管理后台的复杂性
- 高级智能问答
- 长周期训练计划
- 社区互动
- 复杂个性化

## 5. 阶段一 Gate 判断

当前已满足进入下一阶段的关键条件：

- 主战场已收敛到单一首批 ASIN
- Top 痛点已明确
- 核心 3 步链路已定义
- 页面角色已初步落地为可评估 mockup
- 最小埋点与数据回收目标已明确

## 6. 对阶段二的直接输入

阶段二 PoC 应优先验证以下事项：

- 扫码能否顺畅进入 Landing Page
- Step 1 是否有效降低打滑疑虑
- Step 2 是否能建立基本安全信任
- Step 3 是否能降低伤门和装错位置担忧
- Feedback 与 Unresolved Reason 能否稳定回收关键反馈数据
