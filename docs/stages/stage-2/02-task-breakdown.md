# 阶段二：任务分解表

## 1. 任务清单

| 模块 | 任务 | 输出 |
|---|---|---|
| 范围确认 | 锁定首批 ASIN、页面范围、语言范围、技术边界 | PoC 范围说明 |
| 前端搭建 | 初始化 Web / PWA 工程，配置路由与基础布局 | 可运行项目骨架 |
| 页面实现 | 落地 Landing、Step1、Step2、Step3、Feedback、Unresolved Reason | 可点击闭环页面 |
| 内容接入 | 用本地 JSON 或表格导出 JSON 承载首批内容 | 可替换内容源 |
| 埋点接入 | 接入关键事件与必要属性 | 可导出事件数据 |
| 部署上线 | 部署到可公开访问环境并生成二维码 | 演示链接 + 二维码 |
| 兼容验证 | iPhone Safari / Android Chrome / 弱网基础测试 | 测试记录 |
| 演示准备 | 录制 3 分钟闭环视频 | 录屏文件 |
| 复盘沉淀 | 整理 PoC 技术说明与 MVP Backlog | 文档交付 |

## 2. 推荐执行顺序

1. 项目骨架与路由
2. 静态页面闭环
3. 内容 JSON 化
4. 埋点接入
5. 部署上线
6. 验证与录屏
7. Backlog 整理

## 3. 最小成功标准

- 能扫码进入
- 能走完 3 步教程
- 能提交 `resolved_status`
- 能提交 `unresolved_reason_submit`
