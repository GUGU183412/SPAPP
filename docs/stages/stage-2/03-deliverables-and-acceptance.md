# 阶段二：交付物与验收标准

## 1. 阶段二交付物

1. 可演示线上链接
2. 对应二维码
3. 3 分钟演示录屏
4. PoC 技术说明
5. MVP 开发 Backlog

## 2. 建议文档输出

### 2.1 PoC 技术说明

建议包含：

- 项目目标
- 技术栈
- 页面结构
- 内容来源方式
- 埋点事件清单
- 部署方式
- 本地启动方式
- 已知限制

### 2.2 演示说明

建议包含：

- 扫码进入 Landing
- 启动教程
- 走完 Step 1 / 2 / 3
- 完成 Feedback
- 提交 Unresolved Reason
- 查看埋点记录

### 2.3 MVP Backlog

建议包含：

- Must fix before MVP
- Should improve for MVP
- Later / Nice to have

## 3. 验收标准

| 验收项 | 标准 |
|---|---|
| 入口可用 | 扫码后可进入正确页面 |
| 链路可用 | Landing 到 Feedback 全链路可走通 |
| 内容可读 | 首批 SKU 内容完整可演示 |
| 数据可回收 | 关键埋点能记录 |
| 数据可导出 | 可查看访问、完成、解决率、未解决原因 |
| 移动端可用 | iPhone Safari / Android Chrome 可正常使用 |

## 4. 阶段二退出条件

满足以下条件即可认为阶段二完成：

- 最小闭环已跑通
- 关键埋点已生效
- 数据具备复盘价值
- MVP 进入开发前的主要问题已被识别并整理为 Backlog
