# 阶段二：PoC 范围定义

## 1. 阶段目标

阶段二的目标不是开发完整产品，而是验证最小闭环能否真实运行，并且数据能够被回收。

## 2. PoC 范围

### 2.1 业务范围

- 仅覆盖首批 ASIN：`B0BXJLTRSH`
- 仅覆盖首批主场景：门带扣安装与安全确认
- 仅覆盖最小链路：Landing -> Step 1 -> Step 2 -> Step 3 -> Feedback -> Unresolved Reason

### 2.2 产品范围

- Landing Page
- Step 1：防呆锁死
- Step 2：安全背书
- Step 3：门板保护与安装位置
- Feedback：是否解决
- Unresolved Reason：未解决原因

### 2.3 当前不做

- 账号体系
- AI 视频诊断
- 社交 / 打卡
- 多 SKU 扩展
- 复杂内容管理后台
- 复杂离线策略

## 3. PoC 验证问题

阶段二需要回答以下问题：

1. 用户能否顺利从扫码进入主链路
2. Step 1 是否能降低“打滑”困惑
3. Step 2 是否能建立基础安全信任
4. Step 3 是否能缓解伤门与装错位置担忧
5. Feedback 与 Unresolved Reason 是否能回收有效数据

## 4. 成功标准

- 可公开访问
- 手机扫码可进入
- 6 个页面可走通
- 关键埋点可记录
- 基础数据可导出
