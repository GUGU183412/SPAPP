# 阶段三前端手工验证清单

## 1. 验证目标

用于验证阶段三 MVP 前端壳是否满足以下目标：

- 目标驱动流程可完整走通
- 推荐、准备、跟练、反馈、下一步闭环可用
- 语音播报、镜像预览、计时与中断恢复可用
- 埋点事件可落本地
- 视觉风格已收敛到当前设计规范

## 2. 验证环境

- 本地启动开发环境或使用已部署链接
- 建议优先用手机浏览器验证
- 建议同时验证桌面端与窄屏宽度
- 如需验证扫码上下文，可使用 URL 参数模拟：
  - `?asin=B0BXJLTRSH&src=qr&campaign=stage3&product=yoga-ball`

## 3. 主流程验证

### 3.1 Entry 进入页

- 打开页面后，默认进入 `entry`
- 页面可见阶段三入口说明、开始按钮、扫码来源信息
- 点击主按钮后进入 `goal`
- 顶部应显示当前上下文 ASIN

### 3.2 Goal 目标选择页

- 未选择目标前，底部主 CTA 不可继续
- 选择任一目标后，主 CTA 可继续
- 点击继续进入 `equipment`
- 点击顶部返回后，应回到 `entry`

### 3.3 Equipment 器材选择页

- 页面应同时支持：
  - 瑜伽球
  - 其他器材
  - 无器材
- 瑜伽球应作为优先选项展示
- 选择“无器材”后，再选其他器材时应正确切换
- 至少选择一个选项后，主 CTA 可继续
- 点击继续进入 `intake`

### 3.4 Intake 快速问答页

- 页面应包含以下最小输入：
  - 训练经验
  - 可训练时长
  - 当前关注身体状态
  - 当前训练节奏偏好
- 缺少任意必填项时，主 CTA 不可继续
- 补齐后，主 CTA 可继续
- 点击继续进入 `recommendation`

### 3.5 Recommendation 推荐页

- 页面应展示推荐方案标题
- 页面应展示推荐原因
- 页面应展示安全提示
- 页面应展示预计时长与动作数量
- 点击继续进入 `prep`

### 3.6 Prep 准备页

- 页面应展示训练前准备清单
- 页面应展示运行选项
- 语音播报开关可切换
- 镜像预览开关可切换
- 点击继续进入 `session`

### 3.7 Session 跟练页

- 进入后应展示：
  - 当前动作标题
  - 动作提示文案
  - 当前倒计时
  - 总进度
  - 暂停/继续按钮
  - 下一步按钮
  - 提前结束按钮
- 计时应自动递减
- 点击暂停后，计时应停止
- 点击继续后，计时应恢复
- 点击下一步后，应进入下一个 step
- 全部 step 完成后，应自动进入 `feedback`

### 3.8 Feedback 反馈页

- 页面应提供多种结果反馈选项
- 未选择反馈结果前，主 CTA 不可继续
- 可填写补充说明
- 选择任一结果后，主 CTA 可继续
- 点击继续进入 `next-step`

### 3.9 Next Step 下一步页

- 页面应展示系统推荐的下一步
- 页面应允许手动切换不同下一步选项
- 结果应覆盖以下典型路径：
  - repeat easier
  - continue track
  - review prep
  - contact support

## 4. 中断与恢复验证

### 4.1 返回链路

- `goal -> entry`
- `equipment -> goal`
- `intake -> equipment`
- `recommendation -> intake`
- `prep -> recommendation`
- `session -> prep`
- `feedback -> session`
- `next-step -> feedback`

验证要求：

- 返回后，之前已选内容仍保留
- 不应出现状态丢失或页面错乱

### 4.2 提前结束

- 在 `session` 点击“提前结束”
- 应直接进入 `feedback`
- 默认结果应为“中途停止”语义
- 可继续完成反馈与下一步闭环

## 5. 语音与镜像验证

### 5.1 语音播报

- 在 `prep` 开启语音播报
- 进入 `session` 后应在 step 切换时触发语音
- 关闭语音播报后，流程仍应正常可用
- 关闭后不应再有语音触发

### 5.2 镜像预览

- 在 `prep` 开启镜像预览
- 若浏览器支持摄像头权限，应能看到前置镜头画面
- 若拒绝权限或设备不支持，应显示失败/不可用提示
- 关闭镜像预览后，不应继续占用视频流

## 6. 推荐逻辑验证

### 6.1 瑜伽球优先

- 目标选择后，如器材包含瑜伽球，应优先命中瑜伽球模板

### 6.2 其他器材兜底

- 不选瑜伽球，只选其他器材时，应尽量命中对应器材模板

### 6.3 无器材兜底

- 仅选“无器材”时，应仍能生成可执行推荐

### 6.4 时长与强度缩放

- 切换 8 / 12 / 18 分钟和不同强度后
- 推荐时长与 step 时长应发生合理变化

## 7. 埋点验证

### 7.1 本地存储位置

- 打开浏览器开发者工具
- 查看 Local Storage
- 确认存在键：`spapp_stage3_events`

### 7.2 关键事件检查

至少应能看到以下事件被记录：

- `entry_open`
- `goal_select`
- `equipment_select`
- `intake_complete`
- `plan_recommend`
- `plan_accept`
- `prep_ready`
- `session_start`
- `timer_use`
- `voice_prompt_use`
- `mirror_mode_use`
- `exercise_step_complete`
- `session_interrupt`
- `session_complete`
- `session_feedback`
- `next_step_route`
- `flow_exit`

### 7.3 埋点结果要求

- 每个事件带有基本上下文
- 不应出现空数组或无法解析的 JSON
- 正常流程与中断流程都能留下可分析事件

## 8. 视觉验收清单

### 8.1 风格统一

- 所有页面字体系统一致
- 标题层级统一使用大字号 `Lexend`
- 正文与说明文案风格统一

### 8.2 颜色纪律

- 核心颜色应以黑、白、绿、红为主
- 不应再出现明显偏暖的杂色主导界面
- 成功状态统一偏绿
- 风险或结束状态统一偏红

### 8.3 分层与边界

- 页面主要通过面层变化区分区块
- 不应出现明显“线框 App”式分区
- 卡片与浮层边界应偏柔和

### 8.4 阴影控制

- 阴影只用于浮层或强调区
- 不应出现过重、过黑的投影

### 8.5 进度骨架

- 右侧结构应表现为纵向流程骨架
- 当前步骤高亮应清晰

## 9. 响应式验证

- 桌面端布局不拥挤
- 平板或中等宽度下模块可正常换行
- 手机端按钮可完整点击
- 手机端底部主 CTA 不应被遮挡
- 手机端文本不应溢出或重叠
