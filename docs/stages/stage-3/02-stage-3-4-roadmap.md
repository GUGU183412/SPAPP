# 阶段三与阶段四路线图

## 1. 路线图目标

本路线图用于把“阶段二 PoC 闭环”升级为“阶段三目标驱动训练 MVP”，并明确阶段四如何做真实验证与商业判断。

## 2. 阶段三目标

阶段三不是继续修教程，而是完成以下 4 件事：

1. 把产品入口从“SKU 安装引导”升级为“目标 + 器材 + 状态”问答入口
2. 把产品输出从“教程页”升级为“首训方案 + 跟练执行”
3. 让用户真正完成第一次训练，而不只是浏览内容
4. 建立后续扩展到镜像跟练、训练历史和商业闭环的基础结构

## 3. 阶段三交付物

- 目标驱动 MVP 信息架构
- 第一版内容模型
- 第一版推荐规则
- 跟练执行页定义
- 语音与计时能力定义
- 镜像跟练模式定义
- 埋点升级方案
- QA 与里程碑说明
- 老板汇报材料

## 4. 阶段三执行节奏

### 4.1 里程碑 M1：MVP 范围冻结

输出：

- 目标范围
- 器材范围
- V1 / V1.5 / V2 边界

要求：

- 明确负责人
- 与项目经理田楚确认后进入下一里程碑

### 4.2 里程碑 M2：信息架构与内容模型确认

输出：

- 页面流
- 内容字段
- 推荐规则

要求：

- 明确内容 Owner 与产品 Owner
- 与项目经理田楚确认后进入下一里程碑

### 4.3 里程碑 M3：首版 MVP 构建

输出：

- 目标选择
- 器材选择
- 快速问答
- 推荐方案
- 视频跟练
- 计时与语音
- 训练反馈

要求：

- 每周必须有阶段性回传结果
- 不允许长时间无输出推进

### 4.4 里程碑 M4：镜像跟练与验证增强

输出：

- 镜像跟练模式
- 更完善的事件回收
- 实机验证记录

要求：

- 明确技术 Owner
- 与项目经理田楚确认阶段四验证方案

## 5. 阶段四目标

阶段四的重点不再只是看“页面是否能走通”，而是看：

- 用户是否愿意完成一次训练
- 用户是否觉得训练建议有帮助
- 用户是否愿意回来继续使用
- 哪些目标 / 器材组合最值得继续做深
- 是否具备售前、售后与增值服务潜力

## 6. 阶段四验证问题

### 6.1 使用价值验证

- 用户是否愿意完成目标选择
- 用户是否愿意接受推荐方案
- 用户是否能完成首训
- 用户是否使用语音和镜像模式
- 用户是否反馈“有帮助”

### 6.2 留存信号验证

- 次日复访率
- 7 日复访率
- 再次开始率
- 下一步建议点击率

### 6.3 商业价值验证

- 扫码入口是否提升售后承接效率
- 非扫码入口是否具备售前吸引力
- 是否适合推荐更多器材或进阶计划
- 是否值得继续投入社区和服务能力

## 7. 治理要求

这是项目执行层面的硬规则：

1. 每个阶段必须明确负责人
2. 每个关键节点必须与项目经理田楚确认
3. 项目推进过程中必须持续反馈进展和想法
4. 分阶段完成任务并及时返回结果，避免长时间无进展

## 8. 推荐周节奏

- 周一 / 周二：确定本周逻辑、范围和负责人
- 周三：整理中期思路并同步
- 周五：输出阶段性结果或风险说明

## 9. 一句话结论

阶段三负责把产品定义和首版 MVP 做对，阶段四负责验证这个 MVP 是否真的值得继续扩张成更强的训练陪练产品。
