# 阶段三：完整 MVP 定义

## 1. MVP 一句话定义

这是一个 **基于用户目标、可用器材和基础状态信息，快速生成首套训练建议，并支持视频跟练、计时、语音播报和镜像对照训练的轻量训练助手**。

## 2. 为什么不再把 MVP 定义成“安装说明工具”

阶段二验证的是“扫码进入 + 安装闭环 + 反馈回收”这条技术路径能跑通，但研究和业务判断已经表明，真正值得投入的产品不是说明书，而是：

- 用户带着问题来
- 系统给出合适的训练建议
- 用户可以被带着完成第一次训练
- 产品持续承接反馈、复访和商业闭环

安装与安全确认仍然重要，但在新的 MVP 中，它们应成为某些训练计划中的“准备模块”，而不是整个产品的唯一主线。

## 3. MVP 服务对象

第一版 MVP 不只服务单一用户，而是服务“多目标、有限器材、需要快速开始”的用户群体。重点包括：

- 久坐舒缓用户
- 肩颈 / 肩部不适启动用户
- 轻康复或安全入门用户
- 入门塑形用户
- 手边器材有限但希望快速开始训练的居家用户

## 4. MVP V1 的核心链路

`进入产品 -> 选择目标 -> 选择可用器材 -> 快速状态问答 -> 推荐今日方案 -> 开始跟练 -> 提交训练反馈 -> 获取下一步建议`

### 4.1 入口形式

- 二维码扫码进入
- 自主直接打开

### 4.2 入口原则

- 扫码是重要入口，但不是唯一入口
- 产品上下文可保留，但不再强制所有用户先进入 SKU 安装教程
- 用户的目标、状态和可用器材优先于单一产品说明逻辑

## 5. MVP V1 必做功能

### 5.1 目标选择

建议支持 6 到 9 个目标入口，但只对少数高价值路径做深度内容：

- 久坐舒缓
- 肩颈 / 肩部不适
- 康复启动
- 增肌塑形
- 减脂减重
- 体能提升
- 运动健康
- 压力释放
- 其他

### 5.2 可用器材选择

第一版支持多选，建议优先覆盖：

- 弹力带
- 门锚
- 瑜伽垫
- 瑜伽球
- 哑铃
- 无器械

### 5.3 快速状态问答

建议控制在 3 到 5 个问题以内：

- 训练经验
- 今日可用时间
- 当前不适或限制
- 偏好强度

### 5.4 规则化方案推荐

第一版不使用完全自由生成的训练计划，而是基于：

- 目标
- 器材
- 状态问答
- 结构化内容模板

输出一套“首训方案”。

### 5.5 跟练执行页

必须包含：

- 标准训练视频或动作媒体
- 当前动作步骤
- 动作计时
- 休息计时
- 总进度
- 暂停 / 继续 / 下一步

### 5.6 语音播报

建议支持基础语音集：

- 准备开始
- 倒计时
- 休息
- 下一步
- 训练完成

### 5.7 训练反馈

训练结束后收集：

- 是否完成
- 是否合适
- 哪一步不确定
- 身体感受如何

### 5.8 下一步建议

系统根据反馈提供：

- 重复更轻版本
- 继续下一套
- 回看准备模块
- 进入支持路径

## 6. MVP V1.5 推荐增强项

以下能力建议尽快评估并纳入 V1.5：

- 镜像跟练模式
- 简单训练历史
- 再次开始能力
- 更强的推荐解释文案

## 7. 明确后置到 V2 的能力

- 自动动作纠错
- 自动计次
- 姿态识别
- 复杂长期计划
- 社区互动
- 完整会员 / 订阅体系

## 8. 镜像跟练模式的定位

镜像跟练不是“自动监督”，而是：

- 屏幕中同时展示标准动作和用户自己的摄像头预览
- 帮助用户完成自我对照训练
- 不承诺自动判断动作对错

这一定义既有价值，也更适合 PWA 阶段快速验证。

## 9. MVP V1 的推荐深挖路径

第一版建议只做深 3 条：

1. 久坐舒缓
2. 肩颈 / 肩部不适启动
3. 入门塑形

## 10. 数据回收重点

阶段三 MVP 不再只看教程漏斗，而要重点看：

- 目标选择分布
- 器材选择分布
- 推荐方案接受率
- 跟练开始率
- 训练完成率
- 语音使用率
- 镜像模式使用率
- 训练后“有帮助”反馈率
- 下一步建议点击率

## 11. 商业价值定位

该 MVP 同时服务两类价值：

### 11.1 售前价值

- 帮助用户相信品牌具备专业指导能力
- 通过软件体验反向促进硬件成交

### 11.2 售后价值

- 提升产品使用率
- 降低“不会用/怕做错”带来的流失和不满
- 为后续增购、计划升级和社区能力建立基础

## 12. 一句话结论

阶段三的 MVP 不是“更复杂的说明书”，而是一个 **多目标入口、多器材选择、轻智能推荐、视频跟练、计时语音和镜像对照训练** 的目标驱动训练助手。
