# 阶段三反馈与关键结果确认包

## 每 3 天反馈包

每 3 天固定输出一次，避免方向跑偏。

反馈包内容包括：

1. 当前完成了什么
2. 当前页面流转是否可演示
3. 当前有哪些阻塞
4. 下一阶段 3 天要完成什么
5. 是否存在范围偏移风险

建议反馈口径：

- 当前已完成的页面/模块
- 当前已接入的内容模板
- 当前 build 是否通过
- 当前埋点是否已落
- 当前需要确认的事项

## 关键结果确认包

每完成一个关键结果节点，都需要单独做一次确认，而不是只依赖周期性同步。

关键结果节点建议为：

1. 主流程骨架完成
2. 推荐与训练准备完成
3. 跟练 runtime 完成
4. 反馈与下一步建议完成
5. 上线前版本确认

## 每次关键结果确认需要包含

1. 当前版本链接或本地演示方式
2. 当前对应的功能清单
3. 当前未完成项
4. 当前风险与建议取舍
5. 是否可以进入下一步

## 项目经理确认要求

- 所有关键结果节点完成后，需与项目经理田楚确认
- 若出现范围调整、节奏调整、目标变化，也需单独确认
- 未确认前，不进入下一关键结果阶段

## 当前阶段适用原则

- 阶段三周期压缩为 2~3 周
- 反馈节奏固定为每 3 天一次
- 关键结果节点需额外确认一次
- 目标是保证 MVP 主闭环快速落地，同时控制方向不跑偏
