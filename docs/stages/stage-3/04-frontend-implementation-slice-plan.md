# 阶段三前端实施切片计划

## 目标

在 2~3 周内完成阶段三 MVP 前端主闭环，实现：

- 目标选择
- 器材选择
- 状态问答
- 推荐方案
- 训练准备
- 跟练执行
- 训练反馈
- 下一步建议

同时优先接入瑜伽球训练模板库，并保留其他器材与无器材路径。

## 切片安排

### 切片 1：交互骨架与主流程

目标：

- 替换阶段二教程路由
- 建立阶段三页面顺序
- 打通目标、器材、状态、推荐的主链路

完成标准：

- 用户可从 `entry -> goal -> equipment -> intake -> recommendation` 顺利流转
- CTA 启用/禁用规则明确
- 返回路径与 hash 路由可用

### 切片 2：训练准备与跟练 runtime

目标：

- 接入训练准备页
- 接入跟练执行页
- 完成动作计时、休息计时、暂停/继续、下一步
- 接入基础语音播报和镜像预览入口

完成标准：

- 用户可开始一轮完整首训
- runtime 能完成动作推进与训练结束
- 中途退出逻辑可用

### 切片 3：反馈、下一步建议与数据回收

目标：

- 接入训练反馈页
- 接入下一步建议页
- 补齐阶段三埋点
- 完成上线前构建验证

完成标准：

- 用户能完整提交反馈并获得下一步建议
- 关键事件可写入本地埋点存储
- 生产构建通过

## 内容协同要求

- 瑜伽球模板库作为首批重点内容资产优先维护
- 其他器材与无器材路径保持可选，不收缩产品入口
- 内容字段统一为：目标、器材、适用人群、准备事项、安全提示、步骤、时长、常见问题

## 输出物

- 可运行的阶段三 MVP 前端壳
- 首批瑜伽球模板库
- 阶段三埋点链路
- 反馈与下一步建议闭环
